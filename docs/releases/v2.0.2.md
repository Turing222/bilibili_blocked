# v2.0.2 Release Notes

Date: 2026-06-30

## Summary

This release adds a dedicated promoted video card hiding feature. It targets promoted cards rendered as normal DOM cards with `cm.bilibili.com` tracking links, without expanding the legacy page-cleanup feature and without forcing these cards into the BV-based video rule pipeline.

  - Bilibili search result cards
  - Bilibili video page side cards

## Technical Notes

Promoted video cards observed during investigation did not expose normal `/video/BV...` links. Their stable DOM signal was a promoted tracking link:

```text
https://cm.bilibili.com/...
//cm.bilibili.com/...
```

The implementation scans for these links and hides the nearest known card container:

- `div.feed-card`
- `div.bili-feed-card`
- search result `div.bili-video-card` wrappers
- video page side card containers such as `div.video-page-card-small`

The feature stores the original `display` value before hiding and restores it when the switch or master script switch is disabled.

## Non-Goals

- No API requests are added.
- No BV is synthesized for promoted cards.
- Existing title, UP, tag, partition, comment, and stats rules are not reused for these cards.
- The legacy `hideNonVideoElements_Switch` behavior is not expanded in this release.

## Known Limits

- If Bilibili changes the promoted tracking domain away from `cm.bilibili.com`, the selector list will need to be updated.
- If promoted cards stop using visible `<a href>` links, the detection strategy will need another DOM signal.
